Welcome

This website is the WVU instructor and staff handbook for deploying the xPostForecast full-stack application from local development to Microsoft Azure cloud hosting.

The content is built from all project README files and reorganized into a single step-by-step flow so a professor can follow from initial setup through final production validation.

WVU note: For WVU courses, resource groups are not self-created by project teams. Submit a request to Statler ITS to have resource groups created or updated.

How to use this documentation

If you are new to cloud deployment support, follow these pages in order. Each page explains what to verify, why the step matters, and what success should look like before moving to the next stage.

  1. Read Getting Started for prerequisites and required access.
  2. Use GitHub Setup to configure repo workflows and secrets.
  3. Use Azure Basics to confirm Azure resources and settings.
  4. Follow Deployment Steps from first local run to production.
  5. Use Troubleshooting for common errors and quick fixes.

Coverage from the source READMEs

These guides are built from the project README set and rewritten into a single path for instructors, staff, and first-time reviewers. The goal is to reduce ambiguity during class support and grading.

  • Local project setup with Vite, React, Node.js, Express, and MySQL.
  • Authentication with JWT in HTTP-only cookies and protected routes.
  • Azure MySQL connection using SSL certificate and environment variables.
  • STAC climate data integration through Microsoft Planetary Computer APIs.
  • Cloud deployment via Azure App Service, Azure Static Web Apps, and GitHub Actions.
  • Validation steps, health checks, and production troubleshooting.
